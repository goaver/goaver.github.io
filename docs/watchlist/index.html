Watchlist Search

This resource provides access to the watchlist search and adverse media search functionality.

POST api/watchlist/search#

Performs a watchlist search with the specified search criteria

Request Parameters#

  • groupId (required) - The check group the search is in
  • firstName (optional *) - The individual's first name
  • middleName (optional ) - The individual's middle name
  • lastName (optional *) - The individual's last name
  • businessName (optional *) - The business / organization name to search
  • country (optional) - The country for the individual or business
  • stateOrProvince (optional) - The state for the individual or business
  • fileContent (optional) - Base64 image (JPG or PNG) Data URL of the photo containing the individual's face. When provided, this will perform Visual Watchlist search in addition to the Text Watchlist search (associated fees apply). Information about Data URL can be found here
  • fileName (required if fileContent is provided) - The filename for the file content provided for the Visual Watchlist search
  • categories (optional) The watchlist categories to be searched, if omitted the default is to search based on the group settings.
    • Values:
      • Criminal - Criminal related watchlsits
      • Terrorism - International terrorism watchlists
      • Financial - Financial crimes watchlists
      • FinancialNotices - Financial notice watchlists
      • BorrowerDefaults - Borrower default watchlists
      • Political - Politically exposed persons and related watchlists
      • Medical - Medical license revocation and related watchlists
      • Sexual - Sexual offense related watchlists
      • Gambling - Gambling related offenses and restriction watchlists
*NOTE: Either firstName + lastName or businessName must be provided at a minimum.
Example Request#
{
"groupId":"2d1162b5-d6a8-4936-be84-39ec873b7a60",
"firstName":"Some",
"middleName":"Random",
"lastName":"User",
"businessName":"They Work Here Corp",
"country":"USA",
"stateOrProvince":"CA",
"categories":[
"Criminal",
"Terrorism",
"Financial",
"FinancialNotices",
"BorrowerDefaults",
"Political",
"Medical",
"Sexual",
"Gambling",
"Education"
]
}

Response Parameters#

  • id - The unique watchlist search id that was created
Example Response#
{
"id": "f4d8f910-8a7b-42f0-ba40-2b2c17cb9118"
}

GET api/watchlist/{id}#

Retrieves the search and status information for the specified watchlist search

Request Parameters#

  • [Path] id (required) - The watchlist search id returned from the search call

Response Parameters#

  • id - The unique id of the search
  • groupId - The group of the search
  • organizationId - The organization of the search
  • checkId - The associated check that created the watchlist search (if applicable)
  • status - The status of the search
    • See Get Search Results for values
  • searchDate - The UTC date / time of the search
  • searchType - The type of search (text, visual, text and visual)
    • See Get Search Results for values
  • recheckInterval - The auto-recurring search interval for this search to be performed (days)
    • See Get Search Results for values
  • lastRecheckDate - The UTC date / time of the last recurring search that was done based on this search (if applicable)
  • searchCriteria - The search criteria provided for the search
    • See Create Search for parameters

GET api/watchlist/getbycheckid/{checkId}#

Retrieves the watchlist search that was created as the result of a check enrollment that had a check type of watchlist search associated with it.

Request Parameters#

  • [Path] checkId (required) - The check enrollment that the watchlist search was created from

Response Parameters#

  • See Get Watchlist Search by Id

GET api/watchlist/{id}/results#

Retrieves the results for a specified watchlist

Request Parameters#

  • [Path] id (required) - The watchlist search id returned from the search call

Response Parameters#

  • id - The unique id of the search
  • groupId - The group of the search
  • organizationId - The organization of the search
  • checkId - The associated check that created the watchlist search (if applicable)
  • status - The status of the search
    • Values:
      • Completed - Search completed successfully
      • Failed - Search failed due to missing or invalid search criteria
      • Error - Search could not be completed due to unknown error
  • searchDate - The UTC date / time of the search
  • searchType - The type of search
    • Values:
      • Text - Text watchlists
      • TextAndVisual - Text and Facial Match watchlists
  • recheckInterval - The auto-recurring search interval for this search to be performed (days)
    • Values:
      • 0 - None
      • 1 - Daily
      • 7 - Weekly
      • 30 - Monthly
      • 90 - Quarterly
  • lastRecheckDate - The UTC date / time of the last recurring search that was done based on this search
  • searchCriteria - The search criteria provided for the search
    • See Create Search for parameters
  • watchlistResults - The list of matches found for the search criteria (see example below for data model)
  • adverseMediaResults - The list of adverse media results found for the search criteria (see example below for data model)

GET api/watchlist/{id}/searchedlists#

Retrieves the list of watchlists names and versions that were searched for the results

Request Parameters#

  • [Path] id (required) - The watchlist search id returned from the search call

Response Parameters#

  • name - The name of the list that was searched
  • version - The version of that watchlist that was searched
