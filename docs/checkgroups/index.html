Check Groups

A Check Group is an organizational unit for a collection of Verification Checks and / or Watchlist Searches that also provide the default configuration settings for any entities created within the group. Check groups are created and configured via the Aver portal.

It is important to be aware that check groups have an entity limit of 1M Verification Checks and 1M Watchlist Searches. Monitoring the number of entities created within a given check group is important to ensure you are not close to exceeding the capacity of the group. Best practice for high-volume implementations would be to segment your checks into groups or to create new check groups and roll forward at a known interval (ie: yearly), depending on volume. In most cases this is a non-issue, however for high-volume integrations it is important to be aware of.

Check Type Settings#

Check type settings determine what check types, liveness requirements, and supplemental documents are performed or required for any check created within the group.

Check Types#

  • Document Verification - Identification document verification
  • Facial Recognition Analysis - Added liveness detection by requiring a user selfie to facial match to the identification document
  • E-Mail Verification - E-Mail address to verify for the individual
  • Address Verification - Physical mailing address to verify for the individual
  • Phone Verification - Phone number to verify for the individual
  • Watchlist Search - Automatically perform a watchlist search using verified information from the identification check process
  • Visual Watchlist - Use identification document and selfie to face match against visual databases to determine potential use of an alias
  • Accredited Investor - Verify documentation to determine that the individual is an accredited investor
  • Covid Vaccination Verification - Verify documentation to determine that the individual is COVID-19 vaccinated

Facial Recognition Analysis Options#

  • Enhanced Liveness Capture - Require a second selfie from the individual requiring a randomly requested pose to perform enhanced liveness detection

Additional Required Document(s)#

  • Utility Bill - User must provide image(s) of their utility bill
  • Tax Document - User must provide image(s) of their tax document
  • Bank or Credit Card - User must provide image(s) of their bank or credit card
  • Medical Card - User must provide image(s) of their medical card

Check Watchlist Settings#

The check watchlist settings determine whether or not a watchlist search is automatically performed and associated with the verified identity after the check is complete.

Watchlist Search Categories#

  • Criminal - lists containing criminal activity such as most wanted and fugitive lists
  • Terrorism - lists containing known terrorism incidents foreign and domestic
  • Financial Crimes - lists containing incidents around financial crimes and securities fraud
  • Financial Notices - lists with financial notices
  • Borrower Defaults - lists with borrower defaults such as student loan defaults
  • Political - lists with politically exposed persons and other political sanctions
  • Medical - lists for medicare / medicaid violations and other medical license related sanctions
  • Sexual Offenses - lists containing sexual offenses
  • Gambling and Gaming - lists of individuals barred from gambling and gaming control board bans
  • Education - lists containing education related restrictions

Adverse Media Search#

When enabled,the search will also perform a web search to find published articles with the search criteria for any potential criminal and civil offenses.

Watchlist Search Interval#

Setting a watchlist search interval will automatically re-run the same watchlist search criteria at scheduled intervals (weekly, monthly, quarterly, etc)

Check Warning Settings#

The check warnings configure the criteria in which to display a warning if the verification check or watchlist search fall within the restricted criteria.

NOTE: This feature is only available for plans that support check warnings and advanced API functionality.

Identity Warnings#

  • Identification Document Used Previously - warn if the identification document has been used in a previous check
  • Email Address Used Previously - warn if the e-mail address has been used in a previous check
  • Name and Locale Used Previously - warn if the name and location has been used in a previous check
  • Applicant Underage - warn if the applicant is under the age of 18

Access Location#

  • Accessed over VPN or TOR - warn if the applicant IP address is using a VPN or TOR address

Email Warnings#

  • Email Address Invalid - warn if the e-mail address is invalid or suspect

Address Warnings#

  • Address Invalid or Fraud Reported - warn if the mailing address is invalid or fraud has been reported

Phone Warnings#

  • Phone Invalid or Inactive - warn if the phone number is invalid or inactive
  • VOIP or Prepaid - warn if the phone number is a VOIP or prepaid / disposable number

Watchlist and Adverse Media Warnings#

  • Watchlist Search Hits Found - warn if any watchlist search hits were found if the watchlist search was automatically performed as part of the check
  • Adverse Media Hits Found - warn if any adverse media search hits were found if the adverse media search was automatically performed as part of the check

Geographic Restriction Warnings#

  • Restricted Countries - warn if the identity of the individual resides in a restricted country
  • Restricted US States - warn if the identity of the individual resides in a restricted US State
  • Restricted Canadian Provinces - warn if the identity of the individual resides in a restricted Canadian Province

Overriding Check Group Settings#

Check group settings can be overridden on a per-check basis when creating a check via the API. Refer to Create a Check
